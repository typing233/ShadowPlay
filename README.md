# ShadowPlay - 人格复刻与思想对撞沙盒

基于 WebGPU 技术的完全隐私化本地人格复刻与思想对撞沙盒应用。

## 🎯 项目简介

ShadowPlay 允许用户通过粘贴文本描述来创建虚拟人物的"人格影子"，然后让两个人格进行自动对话交流。整个过程完全在本地运行，保护用户隐私。

### ✨ 核心特性

- **文本分析**: 自动提取关键词、语气助词、性格特质
- **情感分析**: 分析文本的积极/消极/中性情感倾向
- **人格建模**: 基于分析结果生成系统提示词
- **自动对话**: 两个人格可以自动进行多轮对话
- **WebGPU 支持**: 支持浏览器 WebGPU 加速
- **完全本地**: 所有数据仅在本地处理，无需联网

## 🚀 快速开始

### 环境要求

- Python 3.8+
- pip 包管理器
- 现代浏览器（推荐 Chrome/Edge 支持 WebGPU）

### 安装步骤

1. **克隆项目**
```bash
cd ShadowPlay
```

2. **创建虚拟环境（可选但推荐）**
```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或
venv\Scripts\activate     # Windows
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **启动服务**
```bash
cd backend
python main.py
```

5. **访问应用**

打开浏览器访问: `http://localhost:8000`

## 📖 使用指南

### Step 1: 创建人格影子

1. 在 **人物 A** 的文本框中粘贴人物描述文本
2. （可选）输入人物名称
3. 点击 **"分析人格 A"** 按钮
4. 对 **人物 B** 重复上述步骤

### 文本示例

**人物 A（乐观探险家）:**
```
他是一个勇敢而聪明的探险家，总是充满热情地探索未知的世界。他善良、温柔，对朋友非常忠诚。在面对困难时，他从不退缩，总是乐观地寻找解决办法。他常说："冒险才是生活的真谛啊！让我们一起去探索吧！"
```

**人物 B（深思哲学家）:**
```
她是一个内向而深思熟虑的哲学家，总是在思考生命的意义。她聪明、理性，对待问题非常认真。她说话时总是带着淡淡的忧伤，喜欢用"呢"、"吧"等温和的语气助词。她常说："真正的智慧在于认识到自己的无知呢。让我们慢慢思考吧。"
```

### Step 2: 开始对话

1. 调整 **对话设置**:
   - **对话轮数**: 5/10/20轮 或 无限对话
   - **对话间隔**: 1/2/3/5秒
   - **先手**: 人物 A 或 人物 B

2. 点击 **"开始对话"** 按钮

3. 观察两个人格的自动交流

### 控制按钮

- **🚀 开始对话**: 启动自动对话
- **⏸️ 暂停**: 暂停当前对话
- **🗑️ 清空**: 清空所有对话记录

## 🔧 API 接口

### POST /api/analyze

分析文本，生成人格档案。

**请求体:**
```json
{
    "text": "人物描述文本...",
    "name": "人物名称（可选）"
}
```

**响应:**
```json
{
    "name": "人物名称",
    "keywords": ["关键词1", "关键词2"],
    "mood_particles": ["啊", "吧"],
    "traits": ["勇敢", "聪明"],
    "emotion_tones": {
        "positive": 0.6,
        "negative": 0.1,
        "neutral": 0.3
    },
    "speaking_style": "描述说话风格...",
    "system_prompt": "系统提示词..."
}
```

### POST /api/dialogue

生成对话回复。

**请求体:**
```json
{
    "character_a_profile": {...},
    "character_b_profile": {...},
    "last_message": "上一条消息",
    "speaker": "A"
}
```

**响应:**
```json
{
    "message": "生成的回复内容",
    "speaker": "A"
}
```

### GET /api/health

健康检查接口。

**响应:**
```json
{
    "status": "healthy",
    "message": "ShadowPlay 服务正常运行"
}
```

## 🔐 隐私保护

ShadowPlay 采用完全本地化的设计：

1. **所有数据本地处理**: 文本分析和对话生成都在本地完成
2. **无数据上传**: 不会将任何用户数据发送到外部服务器
3. **无数据存储**: 关闭浏览器后，所有数据自动清除
4. **开源透明**: 所有代码公开，用户可以自行审计

## 🌐 WebGPU 支持

ShadowPlay 支持 WebGPU 硬件加速：

- **自动检测**: 启动时自动检测浏览器 WebGPU 支持
- **优雅降级**: 不支持 WebGPU 时自动使用 CPU 计算
- **状态显示**: 页面顶部显示 WebGPU 状态

**启用 WebGPU 的浏览器:**
- Chrome 113+
- Edge 113+
- Safari 技术预览版

## 🐛 常见问题

### Q: 服务启动失败？
A: 请检查 Python 版本（需要 3.8+）和依赖是否正确安装。

### Q: 为什么两个人格的对话看起来不太连贯？
A: 当前版本使用基于规则的对话生成。未来版本可以集成本地 LLM 模型来获得更自然的对话。

### Q: WebGPU 状态显示"不可用"？
A: 您的浏览器可能不支持 WebGPU。请使用 Chrome 113+ 或 Edge 113+。

### Q: 如何停止无限对话？
A: 点击 **"暂停"** 按钮或 **"清空"** 按钮。
